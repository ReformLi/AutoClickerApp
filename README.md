# Android自动点击器应用

这是一个仅限个人使用的Android悬浮式连点器应用，使用Kotlin语言开发，兼容Android Studio。

---

## 1. 概述

一款基于 Android 无障碍服务的自动化连点工具，允许用户通过悬浮窗控制全局模拟点击、滑动和系统按键操作，支持多点顺序循环执行，适用于需要重复操作的场景。

## 2. 系统权限要求

| 权限                           | 用途             |
| ---------------------------- | -------------- |
| `SYSTEM_ALERT_WINDOW`        | 显示全局悬浮窗        |
| `BIND_ACCESSIBILITY_SERVICE` | 模拟点击、滑动、全局按键   |
| `FOREGROUND_SERVICE`         | 保持后台运行，防止被系统杀死 |

> 无障碍服务必须由用户在系统设置中手动开启。

---

## 3. 核心功能模块

### 3.1 悬浮窗主控制面板

**布局**：垂直排列的按钮组 + 一个主操作按钮。

**主按钮功能**：

- **点击**：展开/折叠下方操作按钮区域
- **按住拖动**：移动整个悬浮窗，松手后自动吸附到屏幕左边缘或右边缘

**操作按钮列表**（展开后显示）：

| 按钮    | 功能                         | 特殊行为                 |
| ----- | -------------------------- | -------------------- |
| 启动/停止 | 开始或停止连点循环，图标切换（播放/停止）      | 仅当已添加点击点且无障碍服务开启时可点击 |
| 添加    | 在屏幕中央创建一个新的编号点击点           | –                    |
| 减少    | 删除最后添加的点击点（若为滑动对则整个删除）     | –                    |
| 滑动    | 创建一对滑动起点（A）和终点（B），中间有管道连接  | –                    |
| 设置    | 打开全局设置对话框（循环次数、保存/加载点位）    | –                    |
| 隐藏/显示 | 切换悬浮窗半透明、隐藏所有点击点和管道、置灰无关按钮 | 仅非运行状态可操作            |
| 退出    | 关闭悬浮窗服务并通知主界面              | –                    |

**运行时行为**：

- 悬浮窗变为半透明（alpha=0.5）
- 添加、减少、滑动、设置按钮置灰不可用，启动按钮变为停止图标
- 仅主按钮、退出按钮可操作
- 点击悬浮窗空白区域立即停止连点
- 连点期间主按钮仍可拖动，但点击改为停止（不展开菜单）

**隐藏模式行为**：

- 点击点及管道全部隐藏
- 悬浮窗半透明
- 启动、添加、减少、滑动、设置按钮不可用
- 仅主按钮、退出按钮、隐藏（显示）按钮可用
- 再次点击恢复原样

---

### 3.2 点击点（普通按钮）

**创建**：点击“添加”按钮，屏幕中央生成一个直径为 35dp 的橙色圆形按钮，显示编号。

**交互**：

- **按住拖动**：移动点击点到目标位置
- **长按（500ms）**：打开该点的设置对话框（全局悬浮窗样式）
- **短按**：无响应（仅用于定位）

**设置对话框内容**：

- 按钮名称（只读，显示当前编号）
- 类型选项（单选）：点击、双击、返回键、主页键、打开通知栏（滑动点仅显示“滑动”且不可修改）
- 点击位置（单选）：中间 / 随机
- 延迟（毫秒）— 完成本点所有重复后等待的时间
- 触摸时长（毫秒）— 点击/滑动持续时间，滑动类型下标签显示为“滑动持续时间”
- 重复次数 — 当前点连续执行的次数
- 随机距离（像素）— 点击位置在中心点周围随机偏移的范围
- 取消 / 保存按钮

**删除**：

- 点击“减少”按钮删除最后添加的点击点
- 若为滑动对，同时删除配对点及管道

**参数保存/加载**：

- 所有点击点参数（坐标、类型、延迟、触摸时长、重复次数、随机距离等）可保存为 JSON 文件
- 可通过“打开点击文件”加载并恢复所有点及位置

---

### 3.3 滑动点（管道连接的一对起点/终点）

**创建**：点击“滑动”按钮，屏幕中央生成一对橙色圆形按钮（A1、B1，编号随 pair 递增），两点之间绘制一条淡蓝色半透明直线（宽度固定 12px），管道跟随两点移动。

**交互**：

- **按住拖动任意一端**：修改起点/终点坐标，管道实时拉伸/旋转，保持直线连接
- **长按 A1 或 B1**：打开该滑动对的设置对话框（统一设置起终点参数）
- 起点类型为“滑动起点”，终点为“滑动终点”，设置页面类型固定为“滑动”且不可更改

**管道特性**：

- 直线连接两个圆心，始终可见
- 管道无触摸响应，仅作视觉指示
- 拖动节点时管道自动更新

**删除**：

- 点击“减少”删除滑动对时，同时移除起点、终点视图及管道

---

### 3.4 连点执行逻辑

**启动连点**：

- 若当前为隐藏模式，禁止启动
- 将所有点击点设为半透明（alpha=0.3）并添加 `FLAG_NOT_TOUCHABLE`，使手势穿透
- 按点击点 ID 升序循环执行所有点
- 支持总循环次数限制（可在全局设置中配置，0 表示无限）

**各类型动作实现**：

| 类型    | 执行方式                                          |
| ----- | --------------------------------------------- |
| 点击    | 在目标坐标（含随机偏移）执行触摸，时长由触摸时长决定，重复“重复次数”次，完成后等待延迟  |
| 双击    | 两次快速点击（间隔约 100ms），重复执行指定次数                    |
| 长按    | 同点击，但触摸时长控制长按持续时间                             |
| 滑动    | 从起点中心坐标滑动到终点中心坐标，滑动持续时间由触摸时长控制，重复指定次数，完成后等待延迟 |
| 返回键   | 调用无障碍全局返回动作                                   |
| 主页键   | 调用无障碍全局主页动作                                   |
| 打开通知栏 | 调用无障碍全局打开通知栏动作                                |

**停止连点**：

- 手动点击停止按钮、点击悬浮窗空白区域、通知栏“停止”按钮或应用被划掉均会停止
- 停止后恢复悬浮窗透明度、按钮状态，恢复点击点可见性及可触摸状态

---

### 3.5 全局设置对话框

**入口**：点击悬浮窗“设置”按钮

**内容**：

- 总循环次数编辑框（0 或空表示无限循环）
- 保存当前点击位置（导出 JSON）
- 打开点击文件（导入 JSON）
- 取消 / 确定（仅保存循环次数）

**行为**：

- 对话框为全局悬浮窗样式，居中显示，背景透明，圆角白色卡片
- 点击卡片外部区域可关闭（不保存循环次数）

---

### 3.6 主界面（MainActivity）

**功能**：

- 获取权限按钮（红色未授权，绿色已授权）：依次引导用户开启悬浮窗权限和无障碍服务
- 检查权限按钮：手动刷新权限状态
- 启动/关闭悬浮窗按钮（文字和颜色随服务状态变化）
- 状态文本显示当前悬浮窗运行状态

**广播监听**：

- 当悬浮窗服务被关闭时，收到 `SERVICE_STOPPED` 广播，自动更新界面按钮状态

---

## 4. 数据持久化

- 点击点配置保存为 `/data/data/包名/files/click_points.json`
- 内容包括所有点的 ID、坐标、类型、动作参数、滑动对关联等
- 加载时清除现有所有点并重建视图与管道

---

## 5. 非功能性需求

- 兼容 Android 7.0（API 24）及以上
- 悬浮窗默认吸附左侧边缘，可拖动
- 设置对话框等采用 `TYPE_APPLICATION_OVERLAY`，确保在其他应用之上显示
- 前台服务运行时显示通知，提供停止按钮
- 应用从最近任务划掉时自动停止服务并移除悬浮窗 

### 5.1编译要求

- Android Studio Hedgehog或更高版本
- Android SDK 36
- Kotlin 1.9+
- Gradle 9.1+
